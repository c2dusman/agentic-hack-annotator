## Component Inventory

### Page Heading Block
- `<h1>` — "AnnotatorAI" — 36px / weight 700 / #FFFFFF
- `<p>` subtitle — "Paste any URL. Get an annotated visual. Instantly." — 16px / weight 400 / #CCCCCC
- Source: Brief Section 9 exact copy.

### URL Input Field
- `<label>` — "URL" — 16px / weight 600 / #FFFFFF
- `<input type="url">` — placeholder `https://example.com` — 18px / weight 400 / #FFFFFF on #1A1A1A
- Border idle: 1px solid rgba(255,255,255,0.15) — Border focused: 1px solid #FF2D6B
- Source: Brief Section 9.

### Focus Hint Input Field
- `<label>` — "Focus (optional)" — 16px / weight 600 / #FFFFFF
- `<input type="text">` — placeholder `e.g. How to connect Gmail, Setting up API keys` — 18px / weight 400
- Helper text below: "Leave blank and AI will decide what to annotate" — 14px / weight 400 / #CCCCCC
- Source: Brief Section 9 exact copy.

### Generate Button
- Full-width `<button type="submit">` — label "Generate Image"
- Background: #FF2D6B — Text: #FFFFFF — 18px / weight 600
- Border-radius: 8px — Padding: 16px vertical / 24px horizontal
- Hover state: #E0245E (10% darker) with transition 150ms ease
- Disabled state (during processing): opacity 0.5, cursor not-allowed
- Source: Brief Section 9 ("Generate" button, hot pink background). Hover values are Claude's discretion per CONTEXT.md.

### Loading State
- Replaces button area / appears below form while `fetch` is pending
- CSS spinner (rotating border arc, 40px, #FF2D6B on transparent) centered above status text
- Status text: 16px / weight 400 / #CCCCCC
- Step labels cycle at ~5s intervals via `setInterval`:
  1. "📸 Capturing screenshot..."
  2. "🔍 Analyzing page..."
  3. "✍️ Writing annotations..."
  4. "🎨 Rendering image..."
- After 30s if no response: append " Taking longer than usual..." below current step label — same 14px / #CCCCCC style
- Source: Brief Section 9 exact emoji + label copy. D-01 and D-02 from CONTEXT.md.

### Result Section
- Hidden until success response received
- Generated image: `<img>` full-width within 600px column, border-radius 8px
- "Download Image" button: same styling as Generate button but secondary style — border: 2px solid #FF2D6B, background: transparent, text: #FF2D6B
- Source: Brief Section 9 ("Generated image displayed full-width with a Download Image button below").

### Error State
- Hidden until error response received
- `<div>` with border: 2px solid #FF4444, background: rgba(255,68,68,0.08), border-radius: 8px, padding: 16px
- Error text: 14px / weight 400 / #CCCCCC — stage-specific message from server response
- No retry button — user uses the Generate button again
- Source: Brief Section 9 + D-07 from CONTEXT.md.

## Interaction Contract

| Interaction | Behavior |
|-------------|---------|
| Form submit with empty URL | Show inline validation error below URL field. Do not fire fetch. |
| Form submit with invalid URL | Show inline validation error below URL field. Do not fire fetch. |
| Form submit with valid URL | Disable Generate button, hide any previous error/result, show loading state, POST to /api/generate |
| Loading stage cycling | setInterval every 5000ms advances label through 4 steps, wraps at step 4 |
| 30-second threshold | After 30000ms, append long-running notice below current step label |
| Success response | Clear loading state, display result section with image and Download button, re-enable Generate button |
| Error response | Clear loading state, display error box with server message, re-enable Generate button |
| Download click | `<a href="{imageUrl}" download>` — triggers native browser download |
| Subsequent generate | Hides previous result/error before showing loading state again |

Source: Brief Section 9 JavaScript behavior spec + D-07, D-01, D-02 from CONTEXT.md.
