# Phase 3 — UI Design Contract

> Visual and interaction contract for the server-side card template rendered by Puppeteer.
> Generated by gsd-ui-researcher. Consumed by gsd-planner and gsd-executor.

**Scope note:** Phase 3 produces a single static HTML file (`templates/card.html`) rendered
server-side at 1080×1920 by Puppeteer. There is no browser interaction surface — no buttons,
forms, or states. The "UI" is the output PNG image itself. All design values here describe
the card template's visual contract.

## Spacing Scale

Card-internal spacing (all values are multiples of 4):

| Token | Value | Usage in Card Template |
|-------|-------|------------------------|
| xs | 4px | — not used |
| sm | 8px | `.step-content` flex gap between label and description |
| md | 16px | Title→divider margin (`margin-bottom: 16px`) |
| lg | 24px | Step number↔content gap (`gap: 24px`), focus badge horizontal padding |
| xl | 32px | Subtitle→screenshot margin (`margin-bottom: 48px` uses 2xl), footer top padding (`padding-top: 32px`) |
| 2xl | 48px | Screenshot→steps margin (`margin-bottom: 48px`), subtitle bottom margin, subtitle left margin from divider |
| 3xl | 64px | — not used directly |
| card-padding | 80px top/bottom, 72px left/right | Card outer padding (brief §7 exact values) |
| step-gap | 28px | Gap between step rows in flex column |
| divider | 80px × 4px | Pink horizontal rule below title |

Exceptions:
- Card outer padding: 80px vertical / 72px horizontal (non-8-point-scale values, mandated by brief §7)
- Step gap: 28px (between 24px and 32px; brief §7 discretion, confirmed in RESEARCH.md CSS)
- Screenshot slot: `flex: 0 0 520px` fixed height (see Typography note below)

## Typography

All sizes are card-template pixel values (rendered at 1080px wide by Chromium). These are
not browser rem/em values — they are absolute CSS px values for a fixed-dimension Puppeteer
render target.

| Role | Font Family | Size | Weight | Line Height | Usage |
|------|-------------|------|--------|-------------|-------|
| Display | DM Sans | 72px | 700 (bold) | 1.1 | `.card-title` — main heading |
| Heading | Space Grotesk | 32px | 700 (bold) | auto | `.step-label` — step title |
| Body | DM Sans | 32px | 400 (regular) | 1.4 | `.card-subtitle` — subtitle paragraph |
| Body-sm | DM Sans | 28px | 400 (regular) | 1.4 | `.step-description` — clamped 2-line step body |
| Caption | DM Sans | 24px | 400 (regular) | auto | `.footer` — URL + brand attribution |
| Badge | DM Sans | 28px | 700 (bold) | auto | `.focus-badge` — focus hint pill |

Font weights in use: **400 (regular)** and **700 (bold)** only.

Font size scale (4 sizes): **72px** (Display) / **32px** (Heading + Body) / **28px** (Body-sm + Badge) / **24px** (Caption).

Step number circle (`.step-number`): `font-size: 26px` (within 28px badge/body-sm tier visually), `font-weight: 700` (bold).
Note: `.step-number` font-size 26px is a declared exception — it is contained within a 52×52px circle and does not represent a typography hierarchy role; it is a decorative numeral.

Step description clamp: `-webkit-line-clamp: 2` with `-webkit-box` display. Chromium-specific
prefix is correct here — Puppeteer renders via Chromium. Source: CONTEXT.md D-06, RESEARCH.md Pattern 5.

## Card Layout Dimensions

Fixed render target: **1080px wide × 1920px tall** (9:16 ratio).

| Zone | Height | CSS Constraint |
|------|--------|----------------|
| Card outer padding | 80px top + 80px bottom | `padding: 80px 72px` on `.card` |
| Focus badge (conditional) | ~44px + 32px bottom margin | `display: inline-block; margin-bottom: 32px` |
| Card title | ~79px (72px font × 1.1 lh) | `font-size: 72px; line-height: 1.1` |
| Title bottom margin | 16px | `margin-bottom: 16px` |
| Pink divider | 4px + 24px bottom margin | `height: 4px; margin-bottom: 24px` |
| Subtitle | ~90px (32px × 1.4 × ~2 lines) | `font-size: 32px; margin-bottom: 48px` |
| Screenshot slot | **520px fixed** | `flex: 0 0 520px; margin-bottom: 48px` |
| Steps container | fills remaining height | `flex: 1; overflow: hidden` |
| Footer | ~60px | `margin-top: auto; padding-top: 32px` |

Total non-steps height (with focus badge): ~80+44+32+79+16+4+24+90+48+520+48+60+80 = ~1125px
Remaining for steps (3-5 items): ~795px with badge / ~839px without badge.
At 5 steps × ~28px gap: steps each get ~135-150px of space — sufficient for 2-line clamped description.

Screenshot slot: `flex: 0 0 520px` — non-growing, non-shrinking fixed height.
`object-fit: cover; object-position: top` on the `<img>` achieves top-aligned crop (CONTEXT.md D-01).
Screenshot slot border-radius: `12px` (CONTEXT.md D-02).

## Template Placeholders

All 7 placeholders from brief §7 must be replaced by `render.js` before passing HTML to Puppeteer:

| Placeholder | Type | Source | Notes |
|-------------|------|--------|-------|
| `{{CARD_TITLE}}` | string | `annotationData.cardTitle` | HTML-escaped via `escapeHtml()` |
| `{{CARD_SUBTITLE}}` | string | `annotationData.cardSubtitle` | HTML-escaped |
| `{{SCREENSHOT_BASE64}}` | base64 string | `screenshotBase64` param | Replaced LAST — largest substitution |
| `{{STEPS_HTML}}` | HTML block | `buildStepsHtml(annotationData.steps)` | Each step HTML-escaped internally |
| `{{PAGE_URL}}` | string | `pageUrl` param (4th arg) | HTML-escaped; empty string if omitted |
| `{{FOCUS_LABEL}}` | string | `focus` param | HTML-escaped; empty string if focus is null |
| `{{FOCUS_BADGE_STYLE}}` | CSS string | derived from `focus` | `''` when focus truthy, `'display:none'` when null |

Step HTML block structure (generated per step in `buildStepsHtml()`):

```html
<div class="step">
  <div class="step-number">{step.number}</div>
  <div class="step-content">
    <span class="step-label">{step.label}</span>
    <span class="step-description">{step.description}</span>
  </div>
</div>
```

## Copywriting Contract

This phase renders AI-generated copy — Claude Sonnet writes the text, `render.js` injects it.
The contract here governs what the card displays statically.

| Element | Copy | Source |
|---------|------|--------|
| Card title | AI-generated `cardTitle` (max ~60 chars fits at 72px) | `annotate.js` output |
| Card subtitle | AI-generated `cardSubtitle` (~100-120 chars at 32px, ~2 lines) | `annotate.js` output |
| Step label | AI-generated `stepTitle` per step | `annotate.js` output |
| Step description | AI-generated `stepDescription` — clamped to 2 lines | `annotate.js` output |
| Focus badge | User-supplied focus hint text verbatim | `focus` request param |
| Footer brand | Static: `AnnotatorAI · {pageUrl}` | Hard-coded in template |
| Footer separator | `&middot;` (HTML middle dot) | brief §7 exact copy |

Empty state: Not applicable — `renderCard()` is only called with populated annotation data.
If data is missing, `render.js` should throw before reaching Puppeteer.

Error states visible in card: None — errors are handled upstream before render is called.
If render fails, the Express route returns a JSON error; no error state is embedded in the PNG.

Destructive actions: None — this phase has no user-interactive destructive actions.

## Interaction Contract

This phase has no interactive UI surface (Phase 4 handles the web frontend). The only
"interaction" is the Puppeteer render lifecycle:

| Event | Trigger | Behavior |
|-------|---------|----------|
| Template load | `page.setContent(html, { waitUntil: 'domcontentloaded' })` | DOM parsed, fonts @import initiated |
| Font ready | `await page.evaluate(() => document.fonts.ready)` | Blocks screenshot until all declared fonts loaded |
| Screenshot capture | `page.screenshot({ type: 'png', clip: { x:0, y:0, width:1080, height:1920 } })` | Raw PNG buffer |
| Compression | `sharp(buffer).png({ compressionLevel: 9, palette: false }).toBuffer()` | Final optimized PNG |
| Cleanup | `finally` block: `page.close()` then `browser.close()` | No orphaned Chrome processes |

## Focus Badge States

The focus badge has exactly two visual states:

| State | Condition | CSS |
|-------|-----------|-----|
| Visible | `focus` param is a non-empty string | `style=""` (no inline override; CSS `display: inline-block` applies) |
| Hidden | `focus` param is null or empty | `style="display:none"` (inline override hides element) |

Badge position: above `.card-title`, aligned to left edge (`align-self: flex-start`).
Badge appearance: `background: #FF2D6B`, `color: #FFFFFF`, `border-radius: 100px` (pill shape),
`padding: 8px 24px`, `font-size: 28px`, `font-weight: 700`.

Source: REQUIREMENTS.md FOCUS-05, RESEARCH.md Pattern 5, brief §7.

## Sharp Optimization Contract

| Parameter | Value | Rationale |
|-----------|-------|-----------|
| `compressionLevel` | `9` | Maximum PNG compression — no visible quality loss | CONTEXT.md D-07 |
| `palette` | `false` | Full color depth — card embeds photo screenshot, quantization degrades it | CONTEXT.md D-07 |
| Expected output size | 50–150KB | Per CONTEXT.md D-07 estimate |
| Input | Raw PNG buffer from `page.screenshot()` | |
| Output | Compressed PNG buffer written via `fs.writeFileSync` | |

## Design Decisions Summary

All design decisions are locked from upstream. Claude's discretion was exercised for:

| Decision Area | Value Chosen | Rationale |
|---------------|-------------|-----------|
| Screenshot slot height | `flex: 0 0 520px` | Leaves ~795px for steps (5 steps at ~28px gap = comfortable) |
| Step gap | `28px` | Between `lg` (24px) and `xl` (32px); balances breathing room vs. fitting 5 steps |
| Focus badge font-size | `28px` | Matches step-description size; readable without overwhelming title |
| Footer font-size | `24px` | Visually de-emphasized (muted color + smaller size) |
| Step number circle | `52px × 52px` | Touch-target equivalent; visually anchors each step row |
| `.card-subtitle` font-size | `32px` | Consolidates with `.step-label` size; eliminates 36px size to stay within 4-size typography scale |
| `.step-content` gap | `8px` | Corrected from 6px to nearest multiple of 4; adequate separation between step label and description |

All color values, font families, canvas dimensions, and placeholder format are from brief §7 (locked).
